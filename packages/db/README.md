# @manual.email/db

Shared database contracts for the monorepo, **defined as code with
[Drizzle](https://orm.drizzle.team)**. The relational core lives in **D1**
(SQLite); raw message bytes live in **R2**.

## What's here

| Path                  | Purpose                                                       |
| --------------------- | ------------------------------------------------------------- |
| `src/schema.ts`       | Drizzle schema — the single source of truth (`accounts`, `addresses`, `messages`) + inferred row types |
| `src/keys.ts`         | R2 object-key helpers (`messageBodyKey`, `attachmentKey`, …)   |
| `drizzle.config.ts`   | Drizzle Kit config (sqlite dialect, generate-only)            |
| `migrations/`         | SQL migrations **generated** from the schema                  |

Importing apps get camelCase, fully-typed rows:

```ts
import { messages, type Message, type NewMessage } from "@manual.email/db";
import { messageBodyKey } from "@manual.email/db/keys";
```

## Schema as code

`src/schema.ts` is authoritative. Change it, then regenerate migrations —
never hand-edit files in `migrations/`:

```sh
bun run db:generate   # drizzle-kit generate -> migrations/NNNN_*.sql
```

## Storage model

- **D1** (`DB` binding) — accounts, address resolution, message metadata.
- **R2** (`MESSAGES_BUCKET` binding) — raw RFC822 bodies + attachments,
  located via `messages.r2_key`.

Both bindings are declared in `apps/ingress` and `apps/egress`
`wrangler.jsonc`. **This package owns the migrations**: the schema, the
generated migrations, and a minimal `wrangler.jsonc` (D1 binding +
`migrations_dir`) that the `db:migrate*` scripts apply against.

## Migrations

Generated by Drizzle, applied with the D1-native tooling (so generation needs
no Cloudflare credentials):

```sh
bun run db:generate       # regenerate after a schema change
bun run db:migrate:local  # apply to local D1 (miniflare)
bun run db:migrate:prod   # apply to remote D1 (set the real database_id first)
```

> For `drizzle-kit push`/`studio` against a **remote** D1, add
> `driver: "d1-http"` + `dbCredentials` (account id / database id / token) to
> `drizzle.config.ts`.
